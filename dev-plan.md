# 武陵DevOps开发计划

## Stage 1：基础功能

Repo，Issue，Merge Request，Pipelines（需要容器运行时），Wiki，Insight

## Stage 2：企业级功能和体验优化

Project Setup（类似Azure DevOps的操作逻辑）

Project Dashbaord（看板和Scrum并记录所有BackLog）

Repo Setup（类似GitHub操作逻辑）

Test Plan（类似于Azure DevOps的Test Plan）

Package Registry（npm/PyPI/Cargo）

Artifacts（Release，Docker Registry）

## Stage 3 互联和真正的一站式工作台

Channels（是的，忘掉Discord和钉钉吧）

Workspase（基于内置武陵容器运行时的远程开发容器，与[Logos](https://github.com/zixiao-labs/logos)深度集成，类似于Github Codespase，你的VS Code Server可以扔了）

Audio/Video Channels（是的，你没看错，我们连Zoom和腾讯会议的饭碗都抢，基于WebRTC）

Zed风格的CRDT实时协作（与Logos深度集成，至于Zed…这要看Zed Industries什么时候给我们加扩展API（可能在宇宙毁灭之后都没有🙂），先用我们的PWA吧）

从Github/GitLab导入你的工作或者将工作导出到Github/GitLab

同步式镜像（需要配合Github App）

## Stage 4 AI集成

MCP服务器和Skills可以和Github Copilot，Claude Code CLI，Cursor，Zed，OpenAI，Codex一起工作

内置AI自动Triage和Code Review（需要自行提供API Key，目前只支持OpenAI，Anthropic，DeepSeek的API）
